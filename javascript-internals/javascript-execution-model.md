인터프리터 언어
===
인터프리터 언어나 스크립트 언어는 대개 위에서 아래로 한 줄씩 코드가 실행되는 방식으로 만들어지고 실행이 시작되기 전에 거치는 사전 단계가 없음.   
해당 언어로 작성된 프로그램에서는 5번째 줄에 오류가 있다면 4번째 줄이 실행되기 전까지는 오류를 발견하지 못함.  

컴파일 언어
===
인터프리터 언어와 달리 프로그램이 실행되기 전 파싱이라는 사전 단계를 거침.   
파싱은 코드를 추상 구문 트리(AST)로 변환하는 것인데, 만약 코드에 오류가 있다면 이 과정에서 오류를 발견함.  
파싱이 끝나면 컴파일러를 통해 컴퓨터가 실행할 수 있는 형태로 바꿔주는 작업(컴파일)을 수행함.

추상 구문 트리(AST)
---
추상 구문 트리는 소스 코드를 구조화된 트리 형태로 나타낸 것임.
- 추상(Abstract): 모든 구문 세부 정보(괄호, 공백 등)는 제거하고 의미 중심으로 표현
- 구문 트리(Syntax Tree): 프로그램의 문법 구조를 트리 형태로 표현

JS 실행 과정
===
JS로 작성된 소스 코드는 실행 전에 파싱을 거치고, 컴파일러를 통해 최적화된 이진 코드로 변환된 후 실행됨.  
실제 작동 절차는 아래와 같음.
```
    1. 개발자의 손을 떠난 코드는 바벨과 같은 트랜스파일러가 트랜스파일링 후, 웹팩을 비롯한 번들러를 거쳐 번들링되어 그 결과가 JS 엔진에 전달됨.
    2. JS 엔진은 전달받은 코드를 파싱해 추상 구문 트리로 변환.
    3. 추상 구문 트리를 이진 바이트 코드로 변환. 해당 과정에서 JIT 컴파일러가 작동하며 최적화
    4. JS 가상 머신이 프로그램을 실행
```

V8 엔진 구조
---
```
    JavaScript Source
            ↓
          Parser
            ↓
           AST
            ↓
        Ignition (Interpreter)
            ↓
        Bytecode
            ↓
    TurboFan (JIT Compiler)
            ↓
        Machine Code
```

### Ignition (Interpreter)
```
    - AST → 바이트코드(Bytecode)로 변환
    - 바이트코드(Bytecode) 실행
    (바이트코드를 해석 후 CPU가 실행 가능한 최소 명령어 호출)
```

### Bytecode
```
    - 추상적인 가상 머신(VM)이 이해할 수 있는 중간 코드
    - CPU 독립적이므로 모든 플랫폼에서 동일하게 실행 가능
    - 인터프리터가 바로 실행할 수 있도록 변환된 코드
```
```
    // 바이트코드 예시:
    LdaConstant 10
    Star r0
    LdaConstant 20
    Star r1
    Add r0, r1
    Return
    
    인터프리터(Ignition)가 위 예시를 한 줄씩 읽어서 해석 후,
    어떤 연산을 수행할지 해석(decode)한 다음
    CPU 레지스터/메모리에서 실제 연산 수행
    결과를 다시 스택/컨텍스트에 저장
```

### TurboFan (JIT Compiler)
```
    - 자주 실행되는 코드(Hot Code)를 기계어(Machine Code)로 최적화
    - CPU에서 직접 실행 가능
    - 바이트코드와 달리 최적화된 기계어는 인터프리팅 과정 없이 CPU가 바로 명령어를 읽고 수행함.
```
